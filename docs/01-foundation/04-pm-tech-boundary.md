## 第一节：PM必须懂的3个技术概念（5分钟）

### 1.1 Token：AI产品的成本单位

**为什么重要**：不懂Token，就无法估算成本和控制预算

**核心知识**：
- **1个Token ≈ 0.75个英文单词 ≈ 1个中文字**
- 输入和输出分别计费（输出通常贵3-5倍）
- 例子：一次对话消耗多少Token？
  - 用户输入："帮我写一份产品需求文档"（约10 Token）
  - 模型输出：500字的文档（约700 Token）
  - 总消耗：710 Token

**真实案例**：

某电商公司上线AI商品描述生成功能，没控制Token消耗，单月成本从预算的$500飙升到$8000。

**问题分析**：
- 没限制输入长度，商家上传100页商品手册
- 单次消耗10万Token（输入8万+输出2万）
- 日均1000次调用，月成本失控

**解决方案**：
- 限制输入最多5000 Token（约3500字）
- 输出限制500 Token（约350字）
- 月成本降至$600

**练习题**：

假设你的产品DAU 1万，每人每天使用3次AI功能，平均每次消耗1000 Token（输入500+输出500），使用GPT-4o-mini（$0.15/百万输入，$0.6/百万输出），月成本是多少？

<details>
<summary>点击查看答案</summary>

```
日消耗 = 10,000 × 3 × (500×0.15 + 500×0.6) / 1,000,000
       = 30,000 × (0.000075 + 0.0003) / 1
       = 30,000 × 0.000375
       = $11.25/天

月成本 = $11.25 × 30 = $337.5/月
```

**关键点**：输出Token价格是输入的4倍（$0.6 vs $0.15），所以要重点控制输出长度。
</details>

### 1.2 延迟（Latency）：用户体验的生命线

**为什么重要**：延迟>3秒，用户流失率提升40%

**核心知识**：
- **TTFT（Time to First Token）**：用户感知的"反应速度"
- **总延迟** = TTFT + 生成时间
- **行业标准**：TTFT < 1s（优秀），< 2s（合格），> 3s（不及格）

**真实案例**：

某在线教育公司做AI答疑助手，工程师选用Claude Opus（能力强但慢）。

**用户反馈**：
- "等了5秒才开始回答，以为卡了"
- "还不如自己百度快"
- 满意度评分：3.2/5

**优化过程**：
1. 测试发现：Claude Opus TTFT = 3.2秒
2. 改用GPT-4o-mini：TTFT = 0.8秒
3. 满意度提升至4.5/5

**关键决策点**：

| 场景 | 推荐模型 | 原因 |
|------|---------|------|
| 对话/客服 | GPT-4o-mini / Claude Haiku | 快速响应最重要 |
| 长文档分析 | Claude Sonnet / GPT-4o | 可以接受慢一点，但能力要强 |
| 代码生成 | GPT-4o | 平衡速度和质量 |

**练习题**：

你的产品是AI写作助手，用户输入标题后立即生成文章大纲。工程师提出用Claude Opus（TTFT 3秒）。你会同意吗？为什么？

<details>
<summary>点击查看答案</summary>

**不同意**。理由：
1. 写作助手是高频交互场景，用户期待快速反应
2. TTFT 3秒会让用户以为"卡住了"，体验差
3. 大纲生成不需要最强模型，GPT-4o-mini（TTFT 0.8秒）足够

**建议**：
- 用GPT-4o-mini生成大纲（快）
- 用户确认后，用Claude Sonnet生成正文（慢但质量高）
</details>

### 1.3 幻觉（Hallucination）：AI最大的风险

**为什么重要**：模型会编造事实，高风险场景必须应对

**核心知识**：
- **幻觉率**：5-15%（取决于任务复杂度）
- **高风险场景**：医疗/法律/财务建议、引用来源、数字计算
- **应对策略**：来源引用 + 人工审核 + 免责声明

**真实案例**：

某法律咨询AI，用户问"合同违约怎么办"，模型引用了不存在的法条《合同法第258条》。

**后果**：
- 用户据此打官司，败诉
- 用户起诉公司，索赔10万
- 公司紧急下线功能

**教训**：
1. 法律建议属于高风险场景，必须加免责声明
2. 引用法条必须验证真实性（接入权威法律数据库）
3. 高风险输出必须人工审核

**应对策略对比**：

| 策略 | 成本 | 效果 | 适用场景 |
|------|------|------|---------|
| **免责声明** | 低 | 降低法律风险，但不解决幻觉 | 所有AI功能 |
| **来源引用** | 中 | 用户可验证，提升信任 | 知识问答、内容生成 |
| **人工审核** | 高 | 最可靠，但慢且贵 | 医疗/法律/财务等高风险场景 |
| **RAG（检索增强）** | 中 | 降低幻觉率到1-5% | 知识库问答 |

**练习题**：

你的产品是AI健康助手，用户问"头疼怎么办"，模型建议"吃阿司匹林"。这样可以吗？如何改进？

<details>
<summary>点击查看答案</summary>

**不可以**。问题：
1. 医疗建议属于高风险场景，错误建议可能导致医疗事故
2. 模型不知道用户的过敏史、病史，盲目推荐药物很危险
3. 公司可能承担法律责任

**改进方案**：
1. **短期**（本周）：所有医疗建议下方加免责声明"AI生成，仅供参考，请咨询医生"
2. **中期**（2周）：接入权威医疗知识库（RAG），降低幻觉率
3. **长期**（1个月）：高风险输出（药品推荐）必须经医师审核

**正确回答示例**：
> "头疼可能由多种原因引起（感冒、偏头痛、高血压等）。建议：
> 1. 休息并观察症状
> 2. 如果持续或加重，请就医
> 3. 不要自行服药
> 
> ⚠️ AI生成，仅供参考，请咨询专业医生"
</details>

## 第二节：技术方案评审的5个关键问题（5分钟）

### 为什么PM要学会评审技术方案？

**场景**：技术评审会上，工程师提出方案，你需要判断合理性。

**PM的价值**：
- 不是挑刺，而是从产品视角补充工程师可能忽略的点
- 关注：用户体验、成本、风险、可维护性
- 目标：让方案更符合产品目标

---

### 评审框架：4个维度 + 5个问题

#### 评审维度

```
【维度1】目标匹配度
  → 这个方案能解决核心问题吗？有没有过度设计？

【维度2】用户体验
  → 延迟、准确率、失败处理能满足用户期待吗？

【维度3】成本可控性
  → 成本在预算内吗？有优化空间吗？

【维度4】风险识别
  → 有哪些潜在风险？应对方案是什么？
```

---

### 5个关键问题（根据方案类型选择）

#### 如果是 RAG 方案

**背景知识**：RAG = 检索增强生成，用于知识库问答、客服等场景

**5个问题**：

1. **召回精度**："召回精度预计能到多少？有 Baseline 吗？"
   - **为什么问**：召回精度直接影响答案质量
   - **合格标准**：85%+（低于80%用户体验差）
   - **追问**：如果召回不到相关文档，怎么处理？

2. **延迟影响**："Reranker 会增加多少延迟？用户能接受吗？"
   - **为什么问**：Reranker 提升精度但增加延迟
   - **合格标准**：总延迟 < 3秒
   - **权衡**：如果延迟 > 3秒，考虑去掉 Reranker

3. **更新成本**："知识库更新频率怎样？全量重新 Embedding 成本是多少？"
   - **为什么问**：知识库需要定期更新，成本可能很高
   - **合格标准**：增量更新 < $100/次
   - **追问**：能否支持增量更新？

4. **失败处理**："如果检索不到相关文档，怎么处理？"
   - **为什么问**：避免模型胡乱回答
   - **合格标准**：明确告知用户"未找到相关信息"
   - **追问**：失败率预计多少？

5. **评估方法**："怎么评估 RAG 效果？有测试集吗？"
   - **为什么问**：没有评估就无法优化
   - **合格标准**：有100+条测试样本
   - **追问**：准确率、召回率、F1分数各是多少？

**真实案例**：

某公司做企业知识库问答，工程师提出方案：
- 用 GPT-4o + RAG
- 向量数据库用 Pinecone
- 预计召回精度 70%

**PM 提出的问题**：
1. "召回精度 70% 意味着什么？" → 30%的问题答不对，用户体验差
2. "为什么选 GPT-4o 而不是 GPT-4o-mini？" → 简单查询用 mini 够用且便宜5倍
3. "成本预估是多少？" → 工程师没算，PM 要求给出月成本预估

**结果**：
- 召回精度优化到 85%（增加 Reranker）
- 改用 GPT-4o-mini，成本降低 80%
- 增加失败处理："未找到相关信息，请联系人工客服"

---

#### 如果是 Agent 方案

**背景知识**：Agent = 自主决策的AI，能调用工具、多步推理

**5个问题**：

1. **任务边界**："Agent 能处理哪些类型的任务？哪些不能？"
   - **为什么问**：避免用户期待过高
   - **合格标准**：明确列出能力边界
   - **追问**：超出边界的任务怎么处理？

2. **成本控制**："单次任务平均消耗多少 Token？有预算上限吗？"
   - **为什么问**：Agent 多步推理，Token 消耗可能暴涨
   - **合格标准**：单次任务 < $0.1
   - **追问**：有超时终止机制吗？

3. **失败率**："预计失败率是多少？失败后怎么处理？"
   - **为什么问**：Agent 复杂度高，失败率通常 10-20%
   - **合格标准**：失败率 < 15%，有明确的失败提示
   - **追问**：用户能重试吗？

4. **工具调用**："Agent 会调用哪些工具？工具失败怎么办？"
   - **为什么问**：工具失败会导致整个任务失败
   - **合格标准**：有工具失败的降级方案
   - **追问**：工具调用有权限控制吗？

5. **可解释性**："用户能看到 Agent 的决策过程吗？"
   - **为什么问**：黑盒决策用户不信任
   - **合格标准**：展示关键步骤（如"正在查询数据库..."）
   - **追问**：失败时能看到哪一步出错吗？

**真实案例**：

某公司做旅行规划 Agent，工程师提出方案：
- Agent 自动查询航班、酒店、景点
- 生成完整旅行计划

**PM 提出的问题**：
1. "单次规划消耗多少 Token？" → 工程师测试：平均 5万 Token，成本 $0.5/次
2. "成本太高，能优化吗？" → 改为两阶段：先生成草稿（便宜），用户确认后再详细规划
3. "失败率多少？" → 测试发现 25%，主要是航班 API 超时
4. "失败后怎么办？" → 增加重试机制 + 人工客服兜底

**结果**：
- 成本降至 $0.15/次（两阶段方案）
- 失败率降至 12%（增加重试）
- 用户满意度提升 30%

#### 如果是模型选型

**5个问题**：

1. **能力匹配**："为什么选这个模型？对比过其他模型吗？"
   - **为什么问**：避免盲目选择最贵的模型
   - **合格标准**：有对比测试结果
   - **追问**：在我们的测试集上，各模型表现如何？

2. **成本对比**："这个模型的成本是多少？有更便宜的替代方案吗？"
   - **为什么问**：成本差异可能达到 10 倍
   - **合格标准**：月成本在预算内
   - **追问**：如果用便宜模型，效果差多少？

3. **延迟要求**："延迟能满足用户体验要求吗？（<3s）"
   - **为什么问**：强模型通常更慢
   - **合格标准**：TTFT < 2秒
   - **追问**：能否用流式输出改善体验？

4. **上下文长度**："Context Length 够用吗？处理长文档会截断吗？"
   - **为什么问**：不同模型支持的上下文长度差异大
   - **合格标准**：能处理 95% 的用户输入
   - **追问**：超长文档怎么处理？

5. **备选方案**："如果这个模型不可用（API 故障），有备选方案吗？"
   - **为什么问**：单点依赖风险高
   - **合格标准**：有备用模型
   - **追问**：切换备用模型需要多久？

**模型选型速查表**：

| 模型 | 成本 | 速度 | 能力 | 适用场景 |
|------|------|------|------|---------|
| GPT-4o | 中 | 快 | 强 | 通用场景，平衡性能和成本 |
| GPT-4o-mini | 低 | 很快 | 中 | 简单任务、高并发 |
| Claude Sonnet | 高 | 中 | 很强 | 长文本理解、复杂推理 |
| Claude Haiku | 低 | 很快 | 中 | 简单任务、成本敏感 |

### 技术风险检查清单

在评审时，逐项检查以下风险：

- [ ] **幻觉风险**：高风险输出（医疗/财务/法律）有人工审核吗？
- [ ] **成本失控**：有 Token 预算上限和超时终止机制吗？
- [ ] **数据泄露**：用户数据会进入模型训练吗？（确认模型方的数据使用政策）
- [ ] **提示词注入**：有输入过滤和权限隔离吗？
- [ ] **性能瓶颈**：高并发下会不会崩溃？有压测结果吗？
- [ ] **依赖风险**：依赖的外部 API（模型/工具）挂了怎么办？

## 第三节：实战演练（10分钟）

### 演练1：成本失控诊断

**场景**：
运营反馈 AI 功能成本本月超预算 3 倍，你需要快速定位问题并提出解决方案。

**给定信息**：
- 功能：AI 商品描述生成
- DAU：5000
- 平均每人每天使用 5 次
- 使用模型：GPT-4o
- 预算：$1000/月
- 实际成本：$3200/月

**你的任务**：
1. 计算平均每次调用的成本
2. 列出 3 个可能的原因
3. 提出 2 个优化方案

<details>
<summary>点击查看参考答案</summary>

**1. 平均每次调用成本**：
```
总调用次数 = 5000 × 5 × 30 = 750,000 次/月
平均成本 = $3200 / 750,000 = $0.0043/次
```

**2. 可能的原因**：
- **原因1**：输入过长（商家上传完整商品手册，单次 5 万+ Token）
- **原因2**：输出过长（没限制输出长度，生成 2000+ 字描述）
- **原因3**：模型选型不当（简单商品用 GPT-4o 太贵）

**3. 优化方案**：

**方案1：限制输入输出长度**
- 输入限制：最多 3000 Token（约 2000 字）
- 输出限制：500 Token（约 350 字）
- 预期效果：成本降低 60%，至 $1280/月

**方案2：分级模型策略**
- 简单商品（< 100 字输入）：用 GPT-4o-mini（便宜 5 倍）
- 复杂商品（> 100 字输入）：用 GPT-4o
- 预期效果：70% 商品用 mini，成本降低 50%，至 $1600/月

**推荐**：两个方案结合，成本可降至 $800/月（低于预算）

</details>

### 演练2：用户投诉处理

**场景**：
客服反馈用户投诉 AI 客服答案不准确，满意度从 4.2 降至 3.5。

**给定信息**：
- 功能：企业知识库问答（RAG）
- Bad Case 示例：
  1. 用户问"退货政策"，AI 回答了"发货政策"
  2. 用户问"会员等级"，AI 说"未找到相关信息"（但知识库有）
  3. 用户问"最新活动"，AI 回答了 3 个月前的活动

**你的任务**：
1. 用"AI 不准的 5 种类型"分类这 3 个 Bad Case

  - 增量更新知识库（不用全量重建）

  - 模型：Claude Opus

<summary>点击查看参考答案</summary>

**1. 提出的问题**：

**问题1**：为什么选 Claude Opus 而不是 GPT-4o-mini？
- **追问**：简历优化需要最强模型吗？对比过效果吗？

**问题2**：输出 3000 字会不会太长？用户真的会看完吗？

```

单次成本 = (2000×3 + 4000×15) / 1,000,000
         = (0.006 + 0.06)
         = $0.066

日成本 = 5000 × 2 × $0.066 = $660
月成本 = $660 × 30 = $19,800
```

1. 模型选型过度（简历优化不需要最强模型）

3. 没有成本控制机制

**优化1：换模型**
- 改用 GPT-4o-mini（便宜 10 倍）

**最终方案**：
- 模型：GPT-4o-mini
- 输出：500 字核心建议 + 可选详细版
- 增加缓存机制
- 月成本：$400（降低 98%！）


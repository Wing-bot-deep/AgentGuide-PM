## Transformer：大模型的心脏

2017年 Google 发布的《Attention is All You Need》开启了大模型时代。核心机制：**注意力（Attention）**。

### 注意力机制是什么？

人类阅读"苹果公司发布了新手机"时，"公司"和"手机"之间会建立关联。Transformer 用数学方式模拟这种关联。

**对每个词，模型计算它与句子中其他所有词的相关性得分**，得分越高，该词对理解当前词的帮助越大。

```
句子：["苹果", "公司", "发布", "了", "新", "手机"]

理解"手机"时的注意力权重示意：
苹果(0.4) | 公司(0.1) | 发布(0.3) | 了(0.05) | 新(0.15) | 手机(自身)
```

### 为什么 Transformer 优于 RNN（旧技术）？

| 对比 | RNN | Transformer |
|------|-----|-------------|
| 处理方式 | 逐词顺序处理 | 所有词并行处理 |
| 长距离依赖 | 容易遗忘 | 直接建立关联 |
| 训练速度 | 慢 | 快（可利用 GPU 并行） |

## 大模型的训练过程

### 三个阶段

**阶段 1：预训练（Pre-training）**
- 数据：万亿级 Token（互联网文本、书籍、代码）
- 任务：预测下一个词（"北京是中国的___"）
- 目标：让模型学会语言规律和世界知识
- 成本：数百万到数亿美元

**阶段 2：指令微调（Instruction Fine-tuning / SFT）**
- 数据：人工标注的"指令-回答"对（数十万条）
- 目标：让模型学会"按要求回答"而不只是"续写"
- 成本：远低于预训练

**阶段 3：人类反馈强化学习（RLHF）**
- 人工对比两个回答哪个更好，训练奖励模型
- 用奖励模型引导主模型优化
- 目标：让回答更有帮助、更安全

```
原始互联网文本 → [预训练] → 基座模型
                          ↓
             人工标注对话 → [SFT] → 指令跟随模型
                                   ↓
                人工偏好标注 → [RLHF] → 对齐模型（ChatGPT 类）
```

## 推理：模型怎么"说话"

模型不是"查找答案"，而是**逐词预测概率**：

```
输入：今天天气
预测：很(0.4) | 不(0.2) | 挺(0.15) | ...
选择：很
下一步输入：今天天气很
预测：好(0.5) | 糟(0.15) | 热(0.2) | ...
...
```

### Temperature 参数

控制"随机性"，PM 面试常考：

| Temperature | 效果 | 适用场景 |
|------------|------|---------|
| 0 | 每次输出相同，选最高概率词 | 代码生成、精确查询 |
| 0.7（默认） | 适度多样性 | 通用对话 |
| 1.0+ | 输出更发散、创意更强 | 创意写作、头脑风暴 |

## PM 必须能估算的指标

### Context Window（上下文窗口）
模型"一次能看多长的内容"。超出窗口的内容会被截断。

| 模型 | Context Window |
|------|---------------|
| GPT-3.5 | 16K Token |
| GPT-4o | 128K Token |
| Claude 3.5 Sonnet | 200K Token |
| Gemini 1.5 Pro | 1M Token |

1 Token ≈ 0.75 个英文单词 ≈ 1-2 个中文字

### Token 成本估算

```
一次 API 调用成本 = (输入 Token + 输出 Token) × 单价

GPT-4o 示例：
- 输入：$2.5 / 百万 Token
- 输出：$10 / 百万 Token

1000 次用户查询，平均每次 500 输入 + 300 输出 Token：
= 1000 × (500 × 2.5 + 300 × 10) / 1,000,000
= 1000 × (0.00125 + 0.003) = $4.25
```

## 关键术语速查

| 术语 | 一句话解释 |
|------|----------|
| Token | 模型处理文本的最小单位，约 1 词或 1-2 汉字 |
| Embedding | 把文本转成向量（数字列表），使语义可计算 |
| Fine-tuning | 在预训练模型上用特定数据继续训练 |
| RAG | 检索增强生成：回答前先查外部知识库 |
| Prompt | 输入给模型的文字指令 |
| Hallucination | 模型自信但错误的"编造"输出 |
| Quantization | 压缩模型精度，降低推理成本（4-bit, 8-bit） |
| Inference | 用训练好的模型生成输出（区别于训练） |

## 如何使用这些范文

### 步骤1：选择匹配的范文

- 传统PM转型 → 范文1
- 应届生/转行 → 范文2
- 资深AI PM → 范文3

### 步骤2：替换个人信息

```
姓名、联系方式、公司名、时间线
```

### 步骤3：替换具体经历

**保留结构，替换内容**：
- 保留：STAR结构、量化方式、技术关键词
- 替换：公司名、项目名、具体数字

### 步骤4：针对JD定制

**对比JD要求和简历内容**：
- JD要求RAG经验 → 把RAG相关经历放在最前面
- JD要求成本优化 → 突出成本节省的数字
- JD要求多模态 → 补充多模态相关经历

### 步骤5：自查

使用[简历自查清单](./02-resume-guide#七简历自查清单)检查：
- 一句话定位是否清晰
- 每条经历是否有量化结果
- AI技术关键词是否都能展开说细节
- 格式是否整洁

## 常见问题

### Q1：我的经历和范文不完全匹配怎么办？

**答**：范文是参考模板，不是照抄。重点学习：
- 一句话定位的写法
- 量化结果的呈现方式
- 技术关键词的使用

### Q2：我没有那么多量化数字怎么办？

**答**：参考[简历撰写指南](./02-resume-guide#三工作经历量化的正确方式)的"没有数字时怎么办"部分：
- 用相对变化代替绝对值
- 用规模感替代
- 用行业对比

### Q3：我的项目经历很少怎么办？

**答**：
- 立即开始做项目（用Dify搭建RAG原型，1周可完成）
- 写竞品分析文章（证明行业洞察）
- 参与开源项目（证明技术能力）

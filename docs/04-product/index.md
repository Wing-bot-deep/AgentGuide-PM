## 为什么 AI 产品设计不一样？

| 传统产品 | AI 产品 |
|---------|---------|
| 输出确定，设计 UI 流程 | 输出不确定，设计"期望管理" |
| 功能好坏容易测试 | 质量是连续值，很难说"对/错" |
| 迭代周期：需求→开发→上线 | 迭代周期：Prompt→测试→上线（可以很快） |
| 错误：程序崩溃 | 错误：回答差，但用户可能没意识到 |
| 成本：固定 | 成本：随用量变化，Token 是变量 |

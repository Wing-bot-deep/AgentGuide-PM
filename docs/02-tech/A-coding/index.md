## 为什么 AI PM 需要这些？

你不需要写功能代码，但以下场景会真实发生：

- 工程师发来 PR 让你确认需求是否对齐，你得能看懂改了什么
- 联调时接口报错，你得能读懂是 400 还是 500，是前端问题还是后端问题
- 本地跑 Demo 验证 AI 效果，不能每次都等工程师帮你
- 看竞品的 API 文档，判断对方能力边界

**门槛很低，但一旦跨过，与工程师的协作质量会大幅提升。**

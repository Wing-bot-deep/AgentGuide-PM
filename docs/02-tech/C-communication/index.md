## 为什么 AI 产品的沟通更难？

传统产品：需求 → 工程实现，路径相对清晰。

AI 产品的特殊性：

- **效果不确定**：工程师无法承诺 AI 回答的质量，PM 需要和工程师一起定义"成功"
- **Prompt 是产品的一部分**：写 Prompt 的人不一定是写代码的人，但两者必须对齐
- **调参即改需求**：Temperature / 模型版本的变化会影响用户体验，PM 必须参与决策
- **评估比开发难**：很多 AI 功能上线后不知道好不好，需要 PM 提前设计评估方案

**技术沟通能力好的 AI PM，能让工程师少走弯路、让产品迭代更快。**

## 为什么技术理解层是 AI PM 的核心竞争力？

| 场景 | 不懂技术的 PM | 懂技术的 PM |
|------|-------------|------------|
| 工程师说"这个要用 RAG" | 点头说好 | 追问 Chunk 策略和召回率目标 |
| 技术方案评审 | 只能看工期和人力 | 能判断架构合理性和风险点 |
| 写 PRD | 只描述界面和交互 | 能定义模型输入输出和评估标准 |
| 面试被问"你怎么和工程师沟通" | 讲沟通方法论 | 举出读过方案文档、提出技术建议的真实案例 |

## 学习建议

**零基础路线**：A 代码生存 → B AI 技术认知 → C 技术沟通

**有编程基础**：跳过 A，直接 B → C

**面试冲刺**：B 的三篇 + C 的 PRD 模板，覆盖 80% 技术面试题
